## 修复"Attention Is All You Need"等经典论文无法获取相关论文问题

### 问题诊断
1. 问题：经典论文（如"Attention Is All You Need"）无法获取相关论文
2. 原因分析：
   - Semantic Scholar 的 Citations API 返回0个ArXiv ID
   - Semantic Scholar 的 References API 返回0个ArXiv ID
   - 推荐API返回404（可能已弃用）
   - 虽然论文在 Semantic Scholar 有记录，但引用的论文没有关联 ArXiv ID

### 解决方案
1. **移除不可用的推荐API**
   - Semantic Scholar 的 /recommendations 端点对很多论文返回404
   - 已废弃此备用方案

2. **实现 ArXiv 关键词搜索作为主要备用方案**
   - 获取主论文的标题和摘要
   - 使用智能关键词提取算法（过滤停用词，提取重要词汇）
   - 使用 ArXiv API 搜索相关论文
   - 搜索时排除主论文自身
   - 按提交日期排序，获取较新的论文

### 技术改进
1. 新增 `_search_from_arxiv()` 方法
   - 从主论文标题提取关键词（使用预定义的重要词汇列表优先匹配）
   - 构建 ArXiv 搜索查询：`ti:关键词+OR+abs:关键词`
   - 搜索结果按提交日期降序排列

2. 新增 `_extract_keywords()` 方法
   - 包含70+停用词过滤
   - 包含30+重要领域词汇优先匹配（transformer, attention, bert, gpt, llm等）
   - 过滤短词和通用词

3. 改进 `_get_paper_meta()` 方法
   - 同时支持 Semantic Scholar API 和 ArXiv API
   - 获取标题和摘要用于关键词提取

4. 排除主论文自身
   - 搜索结果中过滤掉主论文的 ArXiv ID

### 获取流程（已更新）
1. 从 Semantic Scholar 获取 citations 和 references（优先）
2. 如果获取数量为0，自动切换到 ArXiv 关键词搜索
3. 搜索基于论文标题提取的重要关键词
4. 从搜索结果中随机选择一篇论文返回
5. 排除主论文自身

## 重构相关论文获取逻辑，使用 Semantic Scholar paperId 获取 ArXiv ID

### 问题诊断
用户发现 Semantic Scholar Citations/References API 返回的数据中，虽然有 paperId，但 externalIds 字段中的 ArXiv ID 经常为空或不准确。

**根本原因**：
- API 返回的 `externalIds` 不可靠
- 需要通过 paperId 访问 Semantic Scholar 页面来提取 ArXiv ID

### 新方案

#### 1. 获取流程
1. 通过 ArXiv ID 获取主论文的 Semantic Scholar paperId
2. 使用 paperId 调用 Citations API，获取引用该论文的所有 paperId
3. 使用 paperId 调用 References API，获取该论文引用的所有 paperId
4. 遍历每个 paperId，访问 `https://www.semanticscholar.org/paper/<paperId>` 页面
5. 从页面 HTML 中提取 ArXiv ID
6. 将获取到的 ArXiv ID 保存到主论文目录下的 `related_ids.txt` 文件

#### 2. related_ids.txt 文件格式
```
# 相关论文 ArXiv ID 列表
# citation: 这些论文被主论文引用
# reference: 这些论文引用了主论文

[citation]
arxiv_id_1
arxiv_id_2

[reference]
arxiv_id_3
arxiv_id_4
```

#### 3. 获取相关论文流程
**第一次获取**：
1. 检查 related_ids.txt 是否为空
2. 如果为空，从 Semantic Scholar 获取所有相关论文的 paperId
3. 遍历每个 paperId，访问 Semantic Scholar 页面提取 ArXiv ID
4. 将 ArXiv ID 保存到 related_ids.txt
5. 随机选择一个 ArXiv ID 作为相关论文
6. 下载 PDF，更新 metadata.json，从 related_ids.txt 中移除已使用的 ID

**第二次及以上获取**：
1. 直接从 related_ids.txt 中随机选择一个 ArXiv ID
2. 下载 PDF，更新 metadata.json，从 related_ids.txt 中移除已使用的 ID

#### 4. 删除相关论文流程
1. 删除主论文文件夹中的 PDF 文件
2. 从 metadata.json 中移除该论文记录
3. 将该论文的 ArXiv ID 添加回 related_ids.txt（根据其 relation_type）

## 优化相关论文获取逻辑，按需遍历 Semantic Scholar 页面

### 解决方案

#### 1. 文件结构修改

#### 2. 获取流程优化

1. 从 related_ids.txt 的 [citation] 和 [reference] 中随机选择 paperId

```python
def _search_arxiv_by_title(self, title: str) -> Optional[str]:
    """通过论文标题在 ArXiv 搜索查找 ArXiv ID"""
    # 清理标题，移除特殊字符
    clean_title = re.sub(r'[^\w\s]', ' ', title)
    query = urllib.parse.quote(clean_title[:200])
    search_url = f"{self.arxiv_api_url}?search_query=ti:{query}&max_results=5"
    
    response = self.session.get(search_url, timeout=30)
    if response.status_code == 200:
        feed = feedparser.parse(response.text)
        if feed.entries:
            return match.group(1)  # 返回第一个匹配结果
```

```python
if relation_type == 'reference':
```

```python
if source_section == 'citation':
```

## 修复首次获取相关论文失败问题

### 问题诊断
用户反馈：修改之后，第一次获取主论文的相关论文时总是显示"获取相关论文失败：citation 和 reference 列表都为空"。

**根本原因**：
1. 第一次点击"获取相关论文"时，`related_ids.txt` 文件不存在
2. `_load_ids_file()` 方法返回包含4个空列表的默认字典
3. 代码判断 `citation` 和 `reference` 都为空后直接返回失败
4. 没有去 API 获取相关论文的 arxiv id

### 解决方案
修改 `get_one_related_paper()` 方法的逻辑：

**首次获取（文件不存在）**：
1. 先检查文件是否存在
2. 如果文件不存在，说明是首次获取
3. 调用 `_get_paper_ids_from_api()` 从 API 获取所有相关论文的 ArXiv ID
4. 保存到 `related_ids.txt` 文件
5. 然后从保存的列表中随机选择一篇返回

**非首次获取（文件存在但列表为空）**：
1. 如果文件存在但两列表都为空
2. 说明已经全部获取过或被删除了
3. 直接返回失败，不重新获取

### 修改的代码逻辑
```python
# 获取文件路径
ids_file = self._get_ids_file_path(main_arxiv_id)
file_exists = ids_file.exists()

# 加载已保存的 ID 列表
saved_ids = self._load_ids_file(main_arxiv_id)
pending_citation = saved_ids['citation']
pending_reference = saved_ids['reference']

# 如果待获取列表为空
if not pending_citation and not pending_reference:
    # 如果文件不存在，说明是第一次获取，需要先从 API 获取
    if not file_exists:
        print("首次获取相关论文，从 API 获取...")
        # 从 API 获取相关论文 ID
        api_ids = self._get_paper_ids_from_api(main_arxiv_id)
        # 保存到文件
        saved_ids['citation'] = api_ids.get('citation', [])
        saved_ids['reference'] = api_ids.get('reference', [])
        self._save_ids_file(main_arxiv_id, saved_ids)
        # 更新待获取列表
        pending_citation = saved_ids['citation']
        pending_reference = saved_ids['reference']
    else:
        # 文件存在但列表为空，说明已经全部获取过或被删除了
        print("获取相关论文失败：citation 和 reference 列表都为空")
        return None
```

## 修复 References API 返回0篇论文的问题

### 问题诊断
用户反馈：程序能获取 citation 相关论文，但 reference 论文数始终为0。

**根本原因**：
- Semantic Scholar 的 **References API** 返回的 JSON 字段名是 `citedPaper`，不是 `referencedPaper`
- `_get_reference_papers()` 方法（第237行）使用了错误的字段名 `referencedPaper`
- 导致 `paper` 始终为 `{}`，随后 `paper.get('paperId')` 返回 `None`
- 被 `if not paper.get('paperId'): continue` 跳过，**所有 reference 论文都被跳过**

**对比**：
- Citations API 正确使用字段名 `citingPaper`（第186行）✓
- References API 错误使用字段名 `referencedPaper`（第237行）✗

### 修复方案
将 `_get_reference_papers()` 方法中的字段名从 `referencedPaper` 改为 `citedPaper`：

```python
# 修改前（第237行）
paper = item.get('referencedPaper', {})

# 修改后
paper = item.get('citedPaper', {})
```
